How the bot uses it

Send the key on every request as a Bearer token. Questions are matched by their answer line (no QEMS IDs needed); if a question doesn't exist in the set yet, the item comes back unmatched and the bot should retry it later. Include a unique external_id per item so retries don't record duplicates. 

Base URL: {{ base_url }}
Header:   Authorization: Bearer <your key>

# Validate the key
GET  {{ base_url }}/api/v1/ping

# Record tossup buzzes (who buzzed + optional wrong answer)
POST {{ base_url }}/api/v1/buzzes
{
  "events": [{
    "external_id": "game42-pkt3-q5-user99",
    "answer": "Blaise Pascal",
    "player_name": "someuser#1234",
    "buzz_word_index": 12, "total_words": 45,
    "correct": false, "powered": false, "neg": true,
    "answer_given": "Fermat"
  }]
}

# Record bonus results
POST {{ base_url }}/api/v1/bonus_results
{
  "events": [{
    "external_id": "game42-pkt3-b5-user99",
    "answer": "Bastille",
    "player_name": "someuser#1234",
    "part1_correct": true, "part2_correct": false, "part3_correct": false
  }]
}

# Add comments to a question
POST {{ base_url }}/api/v1/comments
{
  "comments": [{
    "external_id": "game42-pkt3-q5-cmt1",
    "answer": "Blaise Pascal",
    "author_name": "someuser#1234",
    "qtype": "tossup",
    "text": "Second clue might be too easy."
  }]
}
